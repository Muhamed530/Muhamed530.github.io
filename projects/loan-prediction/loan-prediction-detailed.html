üè¶ Can AI Predict Who Will Default on Their Loan?

Every day, financial institutions face a critical question: should they approve this loan application? Behind every decision lies the risk of default ‚Äì borrowers who can't repay what they've borrowed. In 2019 alone, U.S. banks charged off over $130 billion in loans that borrowers couldn't repay. But what if machine learning could help predict these outcomes before they happen?

Using historical data from LendingClub containing over 2.26 million loan records spanning multiple years, I built and compared classification models to predict loan defaults. This project explores whether borrower characteristics and loan features can reliably forecast financial risk, potentially saving lenders millions while ensuring fair access to credit.

üéØ Key Questions Explored:

  • Can we accurately predict which loans will default based on borrower information?
  • Which features are most predictive of loan default risk?
  • How do Random Forest and Logistic Regression algorithms compare for financial risk assessment?
  • What are the ethical implications of using AI in lending decisions?

The Machine Learning Approach: Two Algorithms, One Goal

I implemented and compared two powerful classification algorithms, each with distinct strengths for financial prediction:

üå≤ Random Forest Classifier

Why this algorithm? Random forests excel at capturing complex, non-linear relationships in financial data where interactions between features matter.

  • ‚Ä¢ Ensemble power: Combines 100+ decision trees for robust predictions
  • ‚Ä¢ Feature interactions: Automatically detects relationships between credit score, income, and debt
  • ‚Ä¢ Built-in feature importance: Shows which factors matter most for default prediction
  • ‚Ä¢ Handles missing data: Robust to incomplete borrower information
üìä Logistic Regression

Why this algorithm? Provides interpretable, linear relationships that financial regulators can easily understand and audit.

  • ‚Ä¢ Interpretability: Clear coefficients show how each factor affects default probability
  • ‚Ä¢ Efficiency: Fast training and prediction for real-time lending decisions
  • ‚Ä¢ Probabilistic output: Provides confidence scores for risk assessment
  • ‚Ä¢ Regulatory friendly: Transparent decision-making process

üõ† Technical Report (Process & Methodology)

This technical report documents the complete machine learning pipeline, methodology, and decisions behind the loan default prediction project. It is intended for technical audiences who may want to reproduce, validate, or extend this work.

1. Problem Definition & Approach

Classification Problem: Binary classification to predict loan status (Default vs. Fully Paid)

  • Target Variable: loan_status (engineered to binary: 0 = Fully Paid, 1 = Default)
  • Problem Type: Supervised learning with imbalanced classes
  • Success Metrics: Precision, Recall, F1-Score, ROC-AUC
  • Business Context: Minimize false negatives (missed defaults) while maintaining reasonable approval rates

2. Dataset Overview & Source

  • Source: LendingClub Historical Loan Data (publicly available)
  • Original Size: 2,260,701 records √ó 145 features
  • Time Period: Multi-year historical lending data
  • Data Quality: Real-world financial data with typical missing values and inconsistencies
  • Target Distribution: Imbalanced dataset with more "Fully Paid" than "Default" loans

3. Data Preprocessing & Feature Engineering

3.1 Data Cleaning Steps:

  • Target Engineering:
    • Filtered to loans with clear outcomes (excluded "Current" status)
    • Binary encoding: Default-related statuses ‚Üí 1, Fully Paid ‚Üí 0
  • Missing Value Treatment:
    • Removed features with >50% missing values
    • Applied appropriate imputation strategies for remaining missing data
  • Feature Selection:
    • Selected 7 key features based on domain knowledge and correlation analysis
    • Avoided data leakage by excluding post-loan features

3.2 Selected Features & Rationale:

Feature Type Business Rationale
loan_amnt Numerical Larger loans may carry higher default risk
int_rate Numerical Higher rates indicate higher perceived risk
annual_inc Numerical Income capacity affects repayment ability
dti Numerical Debt-to-income ratio indicates financial stress
fico_range_low Numerical Credit score predicts creditworthiness
grade Categorical LendingClub's risk assessment (A-G)
verification_status Categorical Income verification affects risk profile

3.3 Encoding & Scaling:

  • Categorical Encoding: LabelEncoder for ordinal variables (grade, verification_status)
  • Feature Scaling: StandardScaler applied for Logistic Regression
  • Train-Test Split: 80-20 split with stratification to maintain class balance

4. Exploratory Data Analysis

Key Findings from EDA:

  • Class Imbalance: Approximately 80% fully paid vs 20% default (typical for financial data)
  • Grade Distribution: Strong correlation between LendingClub grades and actual default rates
  • Interest Rate Patterns: Clear relationship between interest rates and default probability
  • FICO Score Impact: Lower FICO scores strongly associated with higher default rates
  • Correlation Analysis: Interest rate and grade highly correlated (by design)

5. Model Implementation & Training

5.1 Random Forest Classifier

Configuration:

  • n_estimators=100 (balance between performance and computational cost)
  • random_state=42 (reproducibility)
  • Default parameters for other hyperparameters

Algorithm Explanation:

Random Forest builds multiple decision trees on random subsets of features and data, then averages their predictions. This ensemble approach reduces overfitting while capturing complex feature interactions crucial for financial risk assessment.

5.2 Logistic Regression

Configuration:

  • max_iter=1000 (ensure convergence)
  • Standardized features (required for logistic regression)
  • Default regularization parameters

Algorithm Explanation:

Logistic regression models the probability of default using a logistic function. It provides interpretable coefficients showing how each feature affects default probability, making it valuable for regulatory compliance and business understanding.

6. Model Evaluation & Results

6.1 Evaluation Metrics Chosen:

  • Accuracy: Overall correctness of predictions
  • Precision: True positives / (True positives + False positives) - Important for minimizing false alarms
  • Recall: True positives / (True positives + False negatives) - Critical for catching actual defaults
  • F1-Score: Harmonic mean of precision and recall - Balances both concerns

6.2 Model Performance Comparison:

Random Forest demonstrated superior performance across all metrics, particularly in capturing the complex relationships between financial features.

6.3 Feature Importance Analysis:

Random Forest feature importance revealed that LendingClub's internal grade assessment was the most predictive feature, followed by interest rate and FICO score. This validates both the platform's risk assessment and the continued relevance of traditional credit metrics.

7. Challenges & Limitations

  • Class Imbalance: Default loans represent minority class, potentially affecting model sensitivity
  • Feature Selection: Limited to 7 features due to computational constraints; more features might improve performance
  • Temporal Factors: Model doesn't account for economic conditions at time of loan origination
  • Missing Data: Some potentially valuable features excluded due to high missing rates
  • Hyperparameter Tuning: Used default parameters; systematic tuning could improve performance

8. Future Improvements
